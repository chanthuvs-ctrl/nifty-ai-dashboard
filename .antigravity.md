# AI Development Guidelines & Constraints

This project uses strict constraints to manage token usage and code stability. All AI assistants must adhere to the following rules at all times.

## Permanent Rules for AI Assistants

1. **Read Documentation First**: Read project documentation (in the `docs/` folder) before analyzing source code.
2. **Minimal Repository Scanning**: Never scan or read the entire repository unless explicitly requested by the user. Only read files necessary for the current task.
3. **Minimize Changes**: 
   - Modify the minimum number of files possible.
   - Modify the minimum number of functions within those files.
   - Prefer extending existing functions rather than rewriting them or introducing new ones.
4. **No Rewriting**: Never rewrite entire files. Use precise search-and-replace tools for contiguous edits.
5. **No Unrelated Refactoring**: Never refactor, clean up, or touch code that is unrelated to the current task.
6. **Preserve Formatting**: Do not change spacing, indentation, line endings, or other formatting style unless it is a direct requirement of the task.
7. **Atomic Commits**: Group edits logically. Implement only one feature or fix per commit. Keep all commits atomic.
8. **Guardrails**: Stop and ask the user for confirmation before modifying more than three (3) files in a single session.

## Response Format Constraint

When completing tasks, your final response to the user must only contain the following sections:

### Files Modified
*List of files modified (with links)*

### Functions Modified
*List of functions modified within those files*

### Summary
*A brief, bulleted explanation of what was changed and why*

### Verification
*The command output or manual testing steps verifying the change*
